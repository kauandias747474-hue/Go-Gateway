# Contributing to Go-Gateway / Contribuindo para o Go-Gateway

## 🇧🇷 Português

Obrigado por se interessar em contribuir para o **Go-Gateway**! Este é um projeto focado em alta performance e engenharia de software rigorosa. Para manter a qualidade do código, pedimos que siga estas diretrizes:

###  Como contribuir?
1. **Reportando Bugs:** Abra uma *Issue* detalhando o comportamento esperado vs. o comportamento atual, incluindo logs e ambiente.
2. **Sugerindo Melhorias:** Se você tem uma ideia para otimizar o *Traffic Shaping* ou o *Circuit Breaker*, abra uma *Issue* para discutirmos a viabilidade técnica antes de codar.
3. **Pull Requests (PRs):**
   - Faça um *Fork* do projeto.
   - Crie uma *Branch* para sua modificação (`git checkout -b feature/nome-da-melhoria`).
   - Garanta que seu código siga os princípios de **Clean Code** e **SOLID**.
   - Se possível, adicione testes unitários para a nova funcionalidade.
   - Abra o PR descrevendo claramente o que foi alterado.

### 🛠️ Padrões Técnicos
- **Linguagem:** Go (versão estável mais recente).
- **Estilo:** Siga as convenções padrão do `go fmt`.
- **Commits:** Use mensagens claras e em inglês (ex: `feat: add prometheus metrics exporter`).

## 🇺🇸 English

Thank you for your interest in contributing to **Go-Gateway**! This project focuses on high performance and rigorous software engineering. To maintain code quality, please follow these guidelines:

###  How to contribute?
1. **Reporting Bugs:** Open an *Issue* detailing the expected vs. actual behavior, including logs and environment specs.
2. **Suggesting Improvements:** If you have an idea to optimize *Traffic Shaping* or the *Circuit Breaker*, open an *Issue* to discuss the technical feasibility before coding.
3. **Pull Requests (PRs):**
   - *Fork* the project.
   - Create a *Branch* for your change (`git checkout -b feature/improvement-name`).
   - Ensure your code follows **Clean Code** and **SOLID** principles.
   - If possible, add unit tests for the new feature.
   - Open the PR clearly describing what was changed.

### 🛠️ Technical Standards
- **Language:** Go (latest stable version).
- **Style:** Follow standard `go fmt` conventions.
- **Commits:** Use clear, concise messages (e.g., `fix: resolve race condition in health probe`).
